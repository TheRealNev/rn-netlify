Liberty Mutual Design System

Nielsen Norman Group defines a Design System as, "A set of standards to manage design at scale by reducing redundancy while creating a shared language and visual consistency across different pages and channels."

Responsibilities

  • Design and maintain components in 4 LMDS Figma libraries.
  • Complete design specification documentation to support the development of LMDS components.
  • Develop LMDS components and features in React.
  • Support maintenance of existing LMDS components in terms of bug-fixes and feature updates in React.
  • Support addition and maintenance of documentation on the LMDS guide site for other designers and developers.

In addition to maintaining what we referred to as the classic style I also had the privilege of designing and building components for something completely fresh. LMDS Dark Mode and LMDS Evolution. Maintaining 4 Figma libraries and supporting them with React was definitely a challenge. However, seeing the end result and making other people's jobs easier is always worth it.

LMDS Guide Site

The LMDS documentation site provides an exhaustive set of design standards and guidance for use with our own components built both in Figma libraries and coded with React and Angular. It not only includes instructions on how to import the styles and components to start using the library, but also guidance on where and when to best utilize them. It contains dos and do nots, text styles, colors, accessibility features and release information not just for the whole library, but for every component in it.

Rite Solutions

I would like to preface this section with a brief explanation;

Unfortunately since the work that I have done here is designated by the United States Government as Classified, I do not have the authority to show the applications. This, of course hinders my ability to fully display my process in a full capacity as a UX/UI designer and engineer.

However, during my time at Rite-Solutions I have worked on two main projects. One being the Inter Subsystem Monitoring Tool (ISMT), and the other a new concept for a cyber security dashboard. A high level description from the ISMT contract itself on sam.gov is as follows

"ISMT is responsible for monitoring all SWFTS Systems of Systems (SOS) interfaces, services, and network health. ISMT alerts operators when a fault or failure has occurred and provides recovery support to the sailor."
-sam.gov

The Cyber Security Dashboard contract description is also available on the Rite-Solutions blog. The other contract was for a new Cyber Dashboard. This is a tool that provides a lot of benefit, and better tracks the cyber health of the submarine, which greatly assists the Commanding Officer and the rest of the sailors. A short description of the Cyber Dashboard's importance by one of the previous program managers at Rite-Solutions is as follows:

"The new Cyber Dashboard and CO Report integrate with the ISMT to assist with cyber readiness, awareness, and recovery. Specifically, the dashboard provides intuitive cyber alerting and management. It identifies real-time, known, and unknown cyber threats in one centralized screen. It provides guidance so the crew, acting like on-board cyber engineers, can eliminate threats and protect their network. It also provides the Commanding Officer with a report of daily and on-demand cyber health status."
-Ken Hartlaub
Program Manager (Rite-Solutions)

When I receive the designs as Sketch Measure files I take a look at the positioning of, and spacing between elements before I start building out a skeleton of the page.
